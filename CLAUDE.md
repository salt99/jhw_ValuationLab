# jhw_ValuationLab — 프로젝트 가이드 (Claude)

가치투자 의사결정 파이프라인을 4개의 독립 웹 도구로 구현한 **개인용 분석 스위트**.
서버리스 · 단일 파일 HTML · vanilla JS · 모바일 우선. GitHub Pages 배포(`salt99.github.io/jhw_ValuationLab/`).

## 먼저 읽을 것 (Source of Truth)

- **`PRD.md`** — 설계 의도·원칙·의사결정 기록의 정본. 작업 전 반드시 정독. 코드와 충돌 시 **코드가 진실**이며 PRD를 갱신한다.
- **`README.md`** — 퀵스타트. ⚠️ 현재 v1 켈리 기준으로 **드리프트**됨(PRD §8). 신뢰하지 말고 코드/PRD 기준으로 판단.

- 새 도구 추가: `index_{name}.html` 생성 → 허브(`index.html`)에 카드 추가(TOOL 0N·액센트색) → PRD §5 키 레지스트리 등록.

## 반드시 지킬 원칙 (PRD §2)

- **P1 출처 검증 (non-negotiable)**: 체크리스트 항목은 거장이 실제로 말한 것으로 검증된 것만. 추론·응용 항목은 "구성" 명시 또는 삭제. *과거 오귀속 3건 발견 이력 있음(§7-D1).*
- **P2 로컬 우선**: 서버·로그인·외부 API 없음. 상태는 전부 `localStorage`, **도구별 고유 프리픽스 필수, 키 공유 금지**(§5). 파일명 변경 = 데이터 마이그레이션 이벤트(경로가 바뀌면 localStorage 분리).
- **P4 사고 보조 ≠ 매매 지시**: 모든 도구에 가정·한계 디스클레이머를 화면 안에 명시. GIGO.
- **P5 검증 후 배포**: 수식·역산 로직은 배포 전 Node.js(DOM-stub)로 단위 검증. 손계산 값과 코드 출력 일치 확인.
- **P6 과공학 경계**: 실사용에서 필요가 확인된 기능만. "있으면 좋은" 기능은 기각.

## 작업 규칙

- **의존성 없음**: 빌드·패키지 매니저 없음. 파일을 브라우저에서 직접 열어 확인. 외부 CDN/프레임워크 추가 지양(오프라인·단일파일 원칙).
- **디자인 토큰 유지**: 다크 테마, 도구별 액센트색(골드/앰버/민트/블루), Pretendard+Mono. PRD §6 참고.
- **수식 변경 시**: 반드시 단위 검증 스크립트로 확인 후 커밋(P5). 밸류에이션·켈리 공식은 PRD §4에 정본.
- **커밋**: 사용자가 요청할 때만. 메시지는 기존 컨벤션(한국어, `{파일명} 업데이트` / 기능 설명)을 따른다.
- **PRD 유지**: 도구 추가·개편 시 PRD §4(명세)·§5(키)·§7(결정 기록) 갱신.
