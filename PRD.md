# jhw_ValuationLab — PRD

> **문서 성격**: 역설계(reverse-engineered) 제품 정의. 배포 코드(2026-07-10 기준 main 브랜치)와 개발 과정의 의사결정 기록에서 추출.
> **목적**: 사람이든 AI든, 이후 작업 세션이 **이 문서 하나로 설계 의도를 이어받는 것**. 기능 카탈로그가 아니라 원칙·결정·제약의 전달이 우선.
> **유지 규칙**: 도구를 추가하거나 크게 개편할 때 §4(명세)·§5(키 레지스트리)·§7(결정 기록)을 갱신한다.

## 1. 제품 정의

**한 줄**: 가치투자 의사결정 파이프라인 — *"판단할 자격이 있는가 → 프리미엄 받을 사업인가 → 숫자로 얼마인가 → 얼마나 살 것인가"* — 를 4개의 독립 웹 도구로 구현한 개인용 분석 스위트.

| 항목 | 내용 |

| 사용자 | 개인 투자자 1인 (집중·장기 투자 실무, Fisher-style) |
| 이론 기반 | Fisher 『Common Stocks and Uncommon Profits』(거인의 어깨 2), Graham, Munger, Lynch, Popper/Soros, Kelly criterion |

## 2. 제품 원칙

**P1. 출처 검증 원칙 (non-negotiable)** — 체크리스트 항목은 거장이 실제로 말한 것으로 검증된 것만 수록한다. 추론·응용으로 구성한 항목은 삭제하거나, 남길 경우 "구성"임을 명시 표기한다. *배경: 개발 중 팩트체크에서 오귀속 3건 발견(§7-D1). 이후 전 도구에 적용되는 최상위 원칙이 됨.*

**P2. 로컬 우선** — 서버·로그인·외부 API 없음. 모든 상태는 `localStorage`. 도구별 키 네임스페이스 분리 필수(§5). 백업은 export/import JSON으로 사용자가 직접.

**P3. 모바일 우선** — 폰 화면 기준 레이아웃, 다크 테마, 홈 화면 추가 전제(`apple-mobile-web-app-capable`). 단일 파일 HTML로 오프라인 캐시 친화.

**P4. 사고 보조 ≠ 매매 지시** — 모든 도구에 가정·한계 디스클레이머를 화면 안에 명시한다. 입력이 추정치면 출력도 추정치(GIGO).

**P5. 검증 후 배포** — 수식·역산 로직은 배포 전 Node.js(DOM-stub)로 단위 검증한다. 대화 중 손계산 값과 코드 출력의 일치를 확인.

**P6. 과공학 경계** — 실사용 흐름에서 필요가 확인된 기능만 추가. "있으면 좋은" 기능은 기각.

**P7. 플랫 파일 규칙** — 모든 도구는 저장소 루트의 `index_{name}.html`. 허브만 `index.html`(GitHub Pages 진입점). 폴더 구조 금지(§7-D2).

## 4. 도구별 명세

### 4.1 TOOL 01 — 능력범위 체크리스트 (`index_circle.html`)

**핵심 질문**: "이 종목을 판단할 자격이 내게 있는가?"

**항목 (6개, 전부 출처 검증됨)**

| # | 항목 | 출처 |
|---|---|---|
| 1 | 전문용어 없이 2분 안에 사업 설명 | 피터 린치 — two-minute drill, 『One Up on Wall Street』 |
| 2 | 5·10·20년 후 이익이 높아질 합리적 확신 | 워런 버핏 — 1996년 주주서한 |
| 3 | 실패 경로를 역방향으로 구성·반박 (인버전) | 찰리 멍거 — "invert, always invert" (야코비 차용) |
| 4 | 사업보고서 원문(DART) 직접 정독 | 벤저민 그레이엄 — 『증권분석』 |
| 5 | 회사 밖 독립 채널 직접 확인 (스커틀벗) | 필립 피셔 — 『Common Stocks and Uncommon Profits』 |
| 6 | 반증 조건을 사전에 명시 (텍스트 입력 포함) | 칼 포퍼 반증주의 → 조지 소로스 |

**판정 로직**: 충족률 84%↑ = 범위 안 / 50~83% = 경계 / 50% 미만 = 범위 밖.
**Veto**: 항목 3(인버전) 미충족 시 점수와 무관하게 최대 "경계" — "범위 안" 등급 불가. *근거: 호감을 분석으로 착각하는 것을 단일 실패점으로 간주(§7-D4).*

**저장**: 종목별 진단 스냅샷 저장/불러오기/삭제. 키: `coc-index`(목록), `coc-record-{id}`(개별).

### 4.2 TOOL 02 — 기업 프리미엄 평가 체크리스트 (`index_premium.html`)

**핵심 질문**: "이 사업이 높은 멀티플을 받을 자격이 있는가, 그리고 시장은 지금 얼마를 주고 있는가?"

**구조**: 프리미엄 자격 11개 요소 평가(거인의 어깨 2 기반; 반복구매·락인·가격결정력 등) → **종합 판단: 자격 × 시장가격의 3×3 매트릭스** → 매수·관망·매도 신호 도출. 크라이시스 시그널(위기 국면 기회 플래그) 포함. 평가 이력 저장.

**저장**: 키 `premiumLab:v1:*`(평가), `premiumLab:hist:*`(이력), `premiumLab:lastTicker`.

*비고: 11개 요소의 서지 정보(판본)는 원저 확인 후 보완 예정 — 코드 내 주석에 동일 취지 기재됨.*

### 4.3 TOOL 03 — 밸류에이션 4-모드 분석기 (`index_roe.html`)

**핵심 아이디어**: PBR · 초과ROE · 지속기간 N · 요구수익률 r 이 잔여이익(Residual Income) 방정식 하나로 묶이므로, **3개를 알면 나머지 1개를 역산**할 수 있다.

**공식** (전액 유보 가정 — 자기자본이 ROE로 복리 성장):

```
적정 PBR* = 1 + Σ[(ROE − r)·(1+ROE)^(t−1)] / (1+r)^t   (t = 1..N)
장기수익률 = [(1+ROE)^N · (청산PBR/진입PBR)]^(1/N) − 1
```

**4개 모드**

| 모드 | 미지수 | 답하는 질문 | 해법 |
|---|---|---|---|
| 1 적정가 | PBR | 정당한 PBR은? | 직접 계산 |
| 2 역산 | ROE | 이 가격이 정당화될 ROE는? | 이분법 |
| 3 역산 | N | 시장이 가정하는 moat 수명은? | 보간 |
| 4 수익률 | r(연수익) | 이 가격에 사면 연 몇 %? | IRR형 (청산 PBR 입력 추가) |

**기본 가정**: 전액 유보(무배당), N년 후 초과수익 소멸(ROE→r), 청산 PBR 기본값 1.0(보수적, §7-D9). ROE=r이면 PBR=1(가치 무차별)이 모델의 앵커.

**저장**: 키 `val4-index`, `val4-rec-{id}`. 모드·입력값 스냅샷 저장.

### 4.4 TOOL 04 — 켈리 · 분할매수 플래너 (`index_kelly.html`)

**핵심 질문**: "각 종목에 얼마를, 어떤 속도로 살 것인가?" — 사이징(켈리) + 실행(분기 분할매수) + 유지보수(재평가)를 한 도구에 통합.

**진화 이력**: v1 "켈리 포트폴리오 배분기"(단순 정규화 배분) → v2 현재 "분할매수 플래너"(전면 재작성, ~106KB). README는 아직 v1 기술(§8).

**켈리 코어**: `f = p/a − q/b` (p 승률 입력, q=1−p 자동; a=하락손실률, b=상승이익률, 현재가 기준). 음수 켈리 종목은 배분 제외(§7-D6). 상대 배분은 풀 켈리 정규화 — 하프 켈리는 정규화 시 약분되어 무의미하므로 총노출 결정에만 사용(§7-D5).

**Method C 보정** (§7-D7): ① 다운사이드 검증 경고 — 하락목표가 현재가의 10% 이내로 얕으면 켈리 출력 신뢰 불가 플래그. ② 개별 포지션 40% 캡 — 초과분은 미캡 종목에 비례 재배분.

**분할매수 실행 계층**:
- 8분기 분할매수 플랜, 분기 순서 강제 — 직전 매수 분기의 바로 다음 분기만 신규 기록 가능. 과거 기록은 값 수정(✎)만 허용, 삭제 후 재입력 불가. *사후확신 편향 차단(§7-D8).*
- 자본 흐름: 날짜 있는 납입 이력. 잔여 페이스 재계산은 미래 분기에만 반영, 과거 매수액은 동결.
- 실적 발표일 추적(earnings date) 보조.

**유지보수 계층**:
- **재평가(📊)**: 원장(ledger) 엔트리로 old→new 값 + 근거(필수) 기록, 켈리 재계산 트리거.
- **비활성 감지**: 재평가 후 켈리가 음수가 된 보유 종목 → 배너로 구분 안내: thesis-broken(논리 훼손 → 즉시 전량 매도 권고) vs valuation-based(밸류에이션 원인 → 분할 매도 허용). 계속 보유 시 근거 기록.

**통화**: USD 기본(미국 브로커리지 전제), KRW는 설정 환율로 참고 표시만.
**백업**: export/import JSON (`dataSnapshot`), 백업 상태 바 UI.
**저장**: 키 `kelly2y` (단일 통합 상태). *구버전 키 `kelly-portfolio`, `kelly-pf-*`는 폐기됨.*

- 액센트: 민트(roe/kelly/hub `--accent:#00d4a0`) vs 골드(circle `--accent:#d9a23b`). 대응 `--accent-dim`·`--accent-ink` 동반.
- 도구 확장: kelly `--cash`/`--cash-dim`, hub `--gold`.
- `.vl-home-btn`(뒤로가기)은 4개 도구에 동일 정의, premium만 자체 스킴(`--border`/`--gold`)으로 대응.

> ⚠️ 토큰을 바꾸면 5개 파일을 각각 수정해야 한다(블록이 동일하므로 복붙은 안전·기계적). 진짜 단일 소스/캐싱을 원하면 외부 파일·빌드가 필요하나 원칙상 기각(§7-D13).
